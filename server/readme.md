# Chat with Document API

FastAPI backend for project-scoped PDF document registration, PostgreSQL metadata storage, and RAG processing.

All resource IDs (`project_id`, `document_id`) are UUID values.

## UploadThing Workflow

The frontend uploads the PDF directly to UploadThing. After UploadThing returns `file_url` and `file_key`, the frontend registers the document with the backend:

```http
POST /documents
Content-Type: application/json
```

```json
{
  "project_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "filename": "contract.pdf",
  "file_url": "https://utfs.io/f/example.pdf",
  "file_key": "uploadthing-file-key"
}
```

The backend stores the document in PostgreSQL with status `processing`, then schedules a background task that downloads the PDF from `file_url` and indexes it for RAG. `file_key` is retained for future file management operations such as deletion or replacement.

You can also register through the project-scoped route:

```http
POST /projects/{project_id}/documents
```

with this JSON body:

```json
{
  "filename": "contract.pdf",
  "file_url": "https://utfs.io/f/example.pdf",
  "file_key": "uploadthing-file-key"
}
```

## Installation

```bash
pip install -r requirements.txt
```

Configure `.env`:

```env
DATABASE_URL=postgresql://user:password@localhost:5432/chat_docs
OPENAI_API_KEY=...
PINECONE_API_KEY=...
PINECONE_INDEX_NAME=chat-docs
```

## Run

```bash
uvicorn main:app --reload
```

## Endpoints

- `POST /projects` creates a project.
- `GET /projects` lists projects.
- `DELETE /projects/{project_id}` deletes a project, all project PDFs, UploadThing files, and Pinecone vectors.
- `POST /documents` registers an UploadThing-hosted PDF for a project.
- `POST /projects/{project_id}/documents` registers an UploadThing-hosted PDF under a path project id.
- `GET /projects/{project_id}/documents` lists project documents.
- `GET /documents/{document_id}` checks processing status.
- `DELETE /documents/{document_id}` deletes one PDF, its UploadThing file, and its Pinecone vectors.
- `GET /projects/{project_id}/messages` lists saved chat messages for a project.
- `POST /projects/{project_id}/chat` asks a question against indexed project documents.

## API Specification
